## Context

`get_market_daily_indicators` 在 `limit` 非 `None` 时直接执行切片。Python 的负索引切片语义会把 `-1`、`-2` 等值解释为"去掉末尾 N 条"，不会报错。这与参数语义"返回条数上限"冲突，也会在 LLM 自动生成参数时导致隐性错误结果。

## Goals / Non-Goals

**Goals:**
- 明确 `limit` 参数契约：仅允许 `None` 或非负整数
- 对非法负值输入快速失败并返回可解释错误
- 保持既有正确路径行为不变（排序、`limit=None`、默认 `limit=50`）

**Non-Goals:**
- 不调整 `sort_by`/`ascending` 语义
- 不改动数据抓取层（`StockFetcher` / provider）
- 不新增分页或范围过滤能力

## Decisions

### 在 MCP Tool 层做参数校验

`limit` 是 MCP API 入参，应在边界层验证并快速失败。这样无需改动数据层接口，也能让错误在最靠近输入的位置被捕获。

### 负值统一抛出 `ValueError`

对 `limit < 0` 直接抛出 `ValueError`，让现有错误处理路径返回 MCP 错误响应。相比隐式修正为 0 或取绝对值，显式报错更可预测，也便于调用方修复请求。

### 为非法输入补充单元测试

新增负值 `limit` 用例，确保不会再出现切片语义导致的"几乎全量返回"问题，并防止后续回归。

## Risks / Trade-offs

- [风险] 更严格校验可能暴露现有调用方的错误参数。  
  → [缓解] 返回明确错误信息，调用方可快速修正；这是预期的契约收敛。
- [权衡] 对非法输入从"返回错误数据"改为"返回错误"，属于行为变更。  
  → [缓解] 在 spec 与测试中明确该行为，保持接口语义一致性。
